# CLF-C02 Domain Map

Source: official CLF-C02 exam guide.

## Domains (weights)
- Cloud Concepts — 24%
- Security & Compliance — 30%
- Cloud Technology & Services — 34%
- Billing, Pricing & Support — 12%

## What I need to be able to explain (plain English)

### Cloud Concepts
- What “cloud” actually means vs on-prem
- Shared Responsibility Model (what AWS handles vs what I handle)
- Basic cloud deployment models (public/private/hybrid) and service models (IaaS/PaaS/SaaS)
- Why organisations use cloud (agility, elasticity, cost model, global reach)

### Security & Compliance
- IAM basics: users vs roles, least privilege, MFA
- Logging/monitoring basics: CloudTrail vs CloudWatch (what each is for)
- Data protection basics: encryption at rest/in transit (KMS conceptually)
- Compliance and governance basics: what “compliance” means in AWS context

### Cloud Technology & Services
- Core services at a high level: EC2, S3, RDS, VPC, Lambda
- Networking basics: VPC, subnets, security groups vs NACLs (just the idea)
- High availability concepts: Regions/AZs, scaling, load balancing (conceptually)

### Billing, Pricing & Support
- Pricing basics: pay-as-you-go, cost allocation tags, free tier concept
- Pricing models: On-Demand vs Reserved vs Spot (conceptual)
- Support plans and what you get
- Basic cost tools: Cost Explorer / Budgets (conceptual)
